# Rembrandt Map 데이터 정제 가이드

> Confluence HTML → Obsidian MD 변환 후 Graph RAG 품질을 높이기 위해 적용한 데이터 정제 방법 전체 정리.
> 적용 순서대로 기술하며, 각 단계의 목적·방법·결과를 포함한다.
> 통합 운영 매뉴얼: `.manual/Graph RAG 데이터 정제 매뉴얼 v3.0.docx`

---

## 1단계 — HTML → MD 변환

**목적:** Confluence에서 추출한 668개 HTML 파일을 Obsidian 호환 마크다운으로 변환

**방법:**
- BeautifulSoup으로 HTML 파싱 → 본문 텍스트 추출
- 테이블 → Markdown 테이블 변환
- 첨부 이미지 경로 → `![[filename.png]]` wikilink 형식으로 변환
- Frontmatter 자동 생성: `title`, `date`, `type`, `status`, `tags`, `source`, `origin`

**결과:** 668개 MD 파일 생성, attachments/ 폴더에 이미지 5,480개 보존

---

## 2단계 — 불필요 데이터 식별 및 정리

**목적:** Graph RAG 노이즈 제거 — 의미 없는 노드가 PageRank와 BFS를 오염시키는 것을 방지

### 삭제한 데이터 (3개)
- **완전 중복 파일:** 내용이 동일하고 ID만 다른 파일 → 하나 삭제

### Archive로 이동한 데이터 (12개)
- **빈 허브 문서:** 제목만 있고 본문이 없는 카테고리 문서
  → 관련 노드가 생기면 나중에 채울 예정이므로 archive 보관

### 보존한 데이터 (133개)
- **내용 있는 archive 문서:** 내용이 있는 구버전 문서는 RAG 맥락 손실 방지를 위해 보존

**결과:** active 393개 → 341개 (정제 + 병합 후)

## 3단계 — Wiki 링크 강화 (enhance_wikilinks.py)

**목적:** 링크가 없는 고립 노드 해소 → BFS 탐색 범위 확대

**방법 3가지:**

### 3-1. 클러스터 링크 주입
- 같은 태그(art/tech/design/level/chief)를 가진 파일들끼리 상호 링크 자동 추가
- 각 파일 하단에 `## 관련 문서` 섹션 생성

### 3-2. 제목 매칭 링크
- 파일 본문 내에 다른 파일의 제목이 텍스트로 등장하면 자동으로 `[[wikilink]]` 변환
- 정규식 look-ahead/behind 로 이미 링크된 텍스트 보호 (placeholder 방식)

### 3-3. Ghost 노드 링크
- 그래프에서 실제 파일 없이 링크만 존재하는 ghost 노드 탐지
- 의미 있는 ghost 노드(`[[아트 파이프라인]]` 등) → 실제 허브 파일로 리다이렉트

**기술적 주의점:**
- 가변 길이 lookbehind는 Python `re` 모듈에서 오류 발생 → `\x00LINK{idx}\x00` placeholder로 기존 링크 보호 후 치환

**결과:** 188개 파일 업데이트, 총 1,782개 링크 생성

---

## 4단계 — 링크 심화 강화 (strengthen_links.py)

**목적:** 3단계 이후에도 링크 없는 파일 170개(43%) 해소 → 0%로 만들기

**방법 5가지:**

### 4-1. 깨진 브래킷 수정
- `enhance_wikilinks.py` 버그로 생긴 `[[[[제목]]` 4중 브래킷 → `[[제목]]` 수정
- 정규식: `re.sub(r'\[{3,}([^\[\]]+?)\]{2}', r'[[\1]]', text)`
- **88개** 파일 수정

### 4-2. 캐릭터 허브 링크 주입
- 캐릭터명이 본문에 등장하는 파일에 해당 캐릭터 허브 문서 링크 자동 추가
- 대상: 다이잔, 스칼렛, 알탄, 보르후, 마투아, 미하일, 카이 등 전 캐릭터
- **340개** 파일에 캐릭터 허브 링크 추가

### 4-3. 계층 부모 링크
- 제목에 상위 카테고리가 포함된 파일(`캐릭터 / 보르후 / 스킬 A`)에 부모 문서 링크 자동 추가
- 계층 구조: `프로젝트 > 캐릭터 > 개별 캐릭터 > 스킬/리소스`

### 4-4. Ghost → Real 허브 교체
- Ghost 노드가 PageRank 상위를 차지하면 RAG 품질 저하 (내용 없는 노드가 허브가 됨)
- 주요 ghost → real 매핑:

| Ghost 노드 | 실제 허브 문서 |
|---|---|
| `[[아트 파이프라인]]` | `[[다이잔_스킬_모음\|아트 파이프라인]]` |
| `[[레벨 디자인]]` | `[[블록 목록_141268286\|레벨 디자인]]` |
| `[[기술 문서]]` | `[[데이터테이블 리스트(서버에 요청)_95174902\|기술 문서]]` |
| `[[게임 디자인]]` | `[[프로젝트A 영웅 스킬 정보_522417784\|게임 디자인]]` |
| `[[이사장 피드백]]` | `[[currentSituation\|이사장 피드백]]` |

### 4-5. 태그 기반 Fallback 링크
- 위 방법으로도 링크가 없는 파일 → 같은 태그의 대표 허브 문서 링크 강제 주입
- 최후 수단으로 고립 노드 완전 해소

**결과:** 341개 파일 업데이트, **링크 없는 파일 0%** 달성
최종 PageRank 상위: 다이잔허브(83), 스칼렛허브(80), 다이잔스킬모음(72), 블록목록(49)

## 5단계 — 문서 병합 (Document Merging)

**목적:** 파편화된 소형 문서들을 통합 → BFS 탐색 시 더 풍부한 컨텍스트 제공

### 5-1. 주간보고 → 월간 통합 (50개 → 12개)
- 50개의 주간업무보고를 월별로 그룹핑하여 12개 월간 통합본 생성
- 원본은 archive로 이동
- 파일명: `주간업무보고_2025-03.md` ~ `주간업무보고_2026-02.md`

### 5-2. 캐릭터 스킬/리소스 통합 (19개 → 4개)
- 캐릭터별 분산된 스킬 문서들을 하나의 통합 파일로 병합

| 병합 결과 파일 | 원본 수 |
|---|---|
| `다이잔_스킬_모음.md` | 7개 |
| `월영_리소스_모음.md` | 5개 |
| `알탄_스킬_모음.md` | 4개 |
| `보르후_스킬_모음.md` | 3개 |

**결과:** 총 69개 문서 → 16개 (문서 수 77% 감소, 정보 밀도 향상)

## 6단계 — 섹션 헤딩 추가

**목적:** Rembrandt Map Passage-level retrieval 활성화 → `##` 헤딩 기준으로 문서를 구간 분리하여 관련 단락만 정밀 추출

**방법:**
- 볼드 텍스트(`**제목**`)가 단락 첫 줄에 있으면 `##` 헤딩으로 승격
- 타입별 기본 섹션 자동 삽입:
  - `spec` 타입 → `## 개요 / ## 기능 명세 / ## 참고사항`
  - `meeting` 타입 → `## 회의 내용 / ## 결정사항 / ## 액션아이템`
  - `decision` 타입 → `## 배경 / ## 결정 내용 / ## 근거`

**결과:** 50개 파일에 `##` 헤딩 추가

---

## 7단계 — BFS 최적화 (문서 앞부분 구조화)

**목적:** Rembrandt Map BFS는 Hop-1 노드에서 첫 1,200자를 가장 중요하게 사용 → 핵심 정보를 문서 상단에 배치

**방법:**
- Frontmatter 직후 `## 개요` 섹션에 핵심 요약 1~3줄 배치
- 상세 내용은 하위 섹션으로 이동
- 테이블이 있는 경우 개요 섹션 바로 아래 배치 (테이블은 정보 밀도가 높음)

---

## 8단계 — PageRank 최적화

**목적:** 의미 있는 허브 문서가 높은 PageRank를 갖도록 인바운드 링크 집중

**원칙:**
- 자주 참조되는 실제 콘텐츠 문서에 링크를 모아야 함
- Ghost 노드(파일 없는 링크)가 허브가 되면 RAG가 빈 노드로 탐색 종료됨

**수행 작업:** 4단계 4-4 Ghost→Real 교체로 해결

**현재 PageRank 상위 5:**
1. `13. 캐릭터 _ 보르후` 허브
2. `07. 캐릭터 _ 스칼렛` 허브
3. `다이잘_스킬_모음`
4. `블록 목록`
5. `currentSituation`

## 10단계 — Obsidian 설정

**목적:** Obsidian이 attachments 폴더를 인식하도록 설정

**문제:** `.obsidian/app.json`이 없으면 이미지 경로를 찾지 못함

**수정:** `.obsidian/app.json` 생성

```json

  "newFileLocation": "folder",
  "newFileFolderPath": "active"
}
```

## 11단계 — Rembrandt Map 이미지 버그 수정 (앱 코드)

**원인:** Obsidian은 `![[attachments/photo.png]]` 형태로 경로 포함 삽입하는데, 코드는 basename만 처리

**문제 흐름:**
- 마크다운 ref: `attachments/photo.png`
- 그래프 노드 ID: `img:attachments/photo.png`
- `imagePathRegistry` 키: `photo.png` (basename만) → 키 미스매치 → 이미지 로드 실패

**수정:**
- `graphBuilder.ts` — `split(/[/\\]/).pop()`으로 basename 추출 후 노드 ID·레이블 생성
- `useVaultLoader.ts` — 프리로드 시 basename fallback으로 레지스트리 조회, 캐시도 basename 키로 저장

## 12단계 — Confluence 신규 배치 처리 (2026-03-04)

**목적:** `confluence/` 폴더에 남아있던 95개 파일을 정제하여 `active_060304/`로 이동 — 노이즈 없는 문서만 선별하고 wikilink 추가

### 12-1. 파일 분류

95개 파일을 3개 카테고리로 수동 분류:

| 카테고리 | 수량 | 기준 |
|---|---|---|
| DISCARD (네비게이션/빈 문서) | ~35개 | 제목만 있고 본문 없음, 월별 진행상황 허브, 개인별 업무현황 페이지 |
| DISCARD (빈 테이블만) | ~29개 | 본문에 헤더만 있는 빈 테이블, 이미지 placeholder만 존재 |
| KEEP+CLEAN | 31개 | 실제 콘텐츠(설명문, 기획서, 가이드) 포함 |

### 12-2. process_confluence.py — Confluence 아티팩트 제거

**처리 내용:**

- **Frontmatter 정제**: `origin`, `confluence_id`, `source` 필드 제거; `date`·`title` 값 내 wikilink 구문 제거 (예: `[[2026_755559|2026]]-03-04` → `2026-03-04`)
- **본문 아티팩트 제거**:
  - `0목차bvicon-list-nested`, `3변경 이력bvicon-history` 네비게이션 줄
  - `> **panel**`, `> **info**`, `> **warning**` 래퍼
  - `> 📎 첨부 이미지:` placeholder
  - `up` 네비게이션 줄
  - 독립 숫자 (`250`, `400` 등 — Confluence 페이지 크기 아티팩트)
  - 이미지 크기 백분율 (`25%25%` 등)
- **빈 테이블 제거**: 모든 셀이 비어있는 테이블 블록 전체 삭제
- **유의미성 필터**: 정제 후 본문 비어있는 파일 (`has_meaningful_content` — 비어있지 않은 줄 ≥2) 자동 제외

**결과:** 64개 DISCARD, 31개 `active_060304/`로 저장

### 12-3. add_links.py — Wikilink 주입

두 가지 방식으로 wikilink 추가:

**LINK_MAP (키워드 → active 파일):**
- 본문에서 알려진 캐릭터명·시스템명 텍스트 발견 시 `[[target|display]]` 형식으로 자동 변환
- 줄당 첫 번째 등장만 링크화; 이미 링크 내부에 있는 텍스트는 보호
- 긴 키워드 우선 처리 (예: `다이잔(Taizan)` → `다이잔` 보다 먼저 처리)

**CONFLUENCE_ID_MAP (구 confluence ID → active 파일명):**
- 이전에 생성된 `[[06._캐릭터_다이잔_1-1._...]]` 형식의 stale wikilink를 현행 active 파일 링크로 교체
- 디스플레이 텍스트: 원래 ID에서 언더스코어를 공백으로 변환 후 말미 숫자 ID 제거

### 12-4. 중복 제거

ID 접미사 매칭으로 `active/` 폴더 기존 파일과 비교 (예: `_863320`):

- `active/`에 더 나은 버전이 있으면 `active_060304/` 버전 삭제
- 비교 기준: 줄 수, 구조적 헤딩 유무, wikilink 존재 여부
